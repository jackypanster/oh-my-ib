# Card 01 — omi brief (one-connection composite snapshot)

Make `tests/brief_command.rs` green: add the read-only `omi brief` subcommand — ONE gateway
connection, six sections (`account_summary`, `pnl`, `pnl_by_position`, `positions`, `orders`,
`executions`) + `account` + `as_of`, one composite JSON — plus the pure assembly seam
`assemble_brief`. Read `arch.md` + `CONTEXT.md` + **ADR 0010 + ADR 0011** (and ADR 0007/0009 they
build on) first.

## Scope (impl-paths only — do NOT touch `tests/brief_command.rs`)

- `src/ib/brief.rs` (new):
  - `pub fn assemble_brief(account: &str, as_of: &str, account_summary: Value, pnl: Value,
    pnl_by_position: Value, positions: Value, orders: Value, executions: Value) -> Value` — the
    pure, frozen seam: one object, exactly the 8 top-level keys
    `{account, as_of, account_summary, pnl, pnl_by_position, positions, orders, executions}`,
    every argument passed through UNMODIFIED (no re-shaping, no key invention).
  - `pub fn brief(cfg: &Config) -> Result<Value, AppError>` — the gateway fn (NOT frozen;
    arch.md §Components, ADR 0010 fixed order, strictly sequential consume-then-drop):
    1. `super::connect(cfg)?` → `super::resolve_account(&client, cfg)?`.
    2. `client.server_time()` → `as_of` string
       `format!("{:04}-{:02}-{:02}T{:02}:{:02}:{:02}Z", t.year(), u8::from(t.month()), t.day(),
       t.hour(), t.minute(), t.second())` (UTC by construction — ADR 0011 §3; name NO type from
       the `time` crate, add NO dependency).
    3. **Consolidated drain** (ADR 0011): ONE `client.account_updates(&account)` pass to
       `AccountUpdate::End`, simultaneously: summary fields via `account.rs`'s accumulator seam;
       `positions` rows via `positions.rs::position_row`; the `(conid, symbol)` discovery list.
       `drop(subscription)` before the next request.
    4. `pnl.rs` with-client seam → take-first (ADR 0007) → 3-key `pnl` object.
    5. `pnl_by_position.rs::sweep_pnl_singles(&client, &account, &discovery)` →
       `shape_pnl_by_position(rows)` (existing pure seam — untouched).
    6. `orders.rs` with-client seam → `all_open_orders` drain-to-End → array; `--account` filter
       semantics verbatim (filter only when `cfg.account` is set — do NOT auto-filter to the
       resolved account; ADR 0011 §Consequences).
    7. `executions.rs` with-client seam → drain-to-End → `merge_executions` array (existing seam).
    8. `Ok(assemble_brief(&account.0, &as_of, ...))`. **Fail-fast** (PRD D3): first error aborts;
       `AppError::data(..., "brief/<section>")` context names the failing section.
- Sibling seam extractions (behavior of the six public commands MUST stay byte-identical):
  - `src/ib/account.rs` — extract the AccountValue accumulator (absorb `(key, value, currency)`;
    emit the 5-key object minus `account`; preserve first-seen-currency-with-NetLiquidation,

  - `src/ib/positions.rs` — extract `pub(crate) fn position_row(&AccountPortfolioValue) -> Value`

  - `src/ib/pnl.rs` — extract a with-client take-first seam returning the 3-key object.
    `pnl(cfg)` unchanged; `pnl_number` untouched.
  - `src/ib/pnl_by_position.rs` — extract `pub(crate) fn sweep_pnl_singles(client, account,
    &[(i32, String)]) -> Result<Vec<PnlSingleRow>, AppError>` (the ADR 0009 loop; take-first via
    `next_data()`, NEVER `iter_data()`; fail-fast naming the conid). `pnl_by_position(cfg)` keeps
    its own discovery drain.
  - `src/ib/orders.rs` / `src/ib/executions.rs` — extract with-client seams returning the bare
    arrays. Public fns unchanged.
