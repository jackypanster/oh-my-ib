# Card 01 — option-chain-default-exchange (client-side `--exchange` filter)

Make `tests/option_chain_filter.rs` green per **ADR 0028**. Read `PRD.md` + `arch.md` + ADR 0028 +
`CONTEXT.md` first — `arch.md` §Data flow + §Component boundaries are the verbatim implementation.
READ-path card: no `trade.rs`, no write gate, no `OMI_ALLOW_LIVE`.

## Scope (impl-paths only — do NOT touch `tests/`)

- `src/ib/option_chain.rs`:
  - NEW pure seam:
    `pub fn filter_chain_rows(rows: Vec<ChainRow>, exchange: &str) -> Vec<ChainRow>`
    — `exchange == ""` ⇒ return `rows` unchanged (passthrough); else
    `rows.into_iter().filter(|r| r.exchange == exchange).collect()` (exact-string, case-sensitive,
    ALL matches kept, input order preserved).
  - Gateway fn `option_chain`: change the reqSecDefOptParams call from `&args.exchange` to `""`
    (`client.option_chain(&args.symbol, "", SecurityType::Stock, conid)`), and insert
    `let rows = filter_chain_rows(rows, &args.exchange);` BETWEEN the drain loop and the final
    `Ok(shape_option_chain(&args.symbol, conid, rows))`.
  - Update the module doc comment (~line 65 "`--exchange` is a server-side passthrough") and the fn
    doc to describe the client-side filter (server call is unfiltered; `--exchange` filters locally).
  - `shape_option_chain` + `ChainRow`: **UNTOUCHED**.
- `src/ib/mod.rs`: add `filter_chain_rows` to the existing
  `pub use option_chain::{option_chain, shape_option_chain, ChainRow};` re-export.
- `src/cli.rs` `OptionChainArgs.exchange`: keep `default_value = "SMART"`; update the `///` doc to
  "Client-side exchange filter; `SMART` (default) = consolidated view, `''` = all exchanges".

## Out of scope

`shape_option_chain` behavior; option-quote; server-side retry/fallback; dedup of identical rows;
drain/timeout posture (ADR 0016/0019 reused verbatim).
