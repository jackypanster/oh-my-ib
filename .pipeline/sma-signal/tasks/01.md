**Pure `pub fn sma_signal(bars: &[Bar], n: usize) -> SmaSignal`** (ADR 0034 D-RULE):

```
bars_used = bars.len()
sma_at(i) = mean(bars[i+1-n ..= i].close)          // defined only when i+1 >= n
if bars.is_empty()            → Insufficient, as_of_month_end=(0,0), numeric 0.0
if bars.len() < n             → Insufficient, as_of_month_end=bars.last().ym, numeric 0.0
last_ym = bars.last().ym
as_of_idx = max i where bars[i].ym < last_ym ,  else bars.len()-1   // exclude in-progress final month
if as_of_idx + 1 < n          → Insufficient, as_of_month_end=bars[as_of_idx].ym, numeric 0.0
sma = sma_at(as_of_idx); month_end_close = bars[as_of_idx].close
state = if month_end_close >= sma { Hold } else { Exit }
distance_pct = (month_end_close - sma)/sma*100
li = bars.len()-1; latest_close = bars[li].close; latest_sma = sma_at(li)   // li+1>=n holds (len>=n)
latest_distance_pct = (latest_close - latest_sma)/latest_sma*100
as_of_month_end = bars[as_of_idx].ym
```

`ym` compares as a plain tuple (`(i32,u32)` lexicographic = chronological). No dates, no ibapi here. Any
NEW f64 comparison in `src/` must use an epsilon (clippy `float_cmp` under `-D warnings`) — but this fn
does no `==` on floats, just `>=`, so it's clean.

**Gateway `pub fn sma_signal_cmd(cfg: &Config, args: &SmaSignalArgs) -> Result<Value, AppError>`**
(review-by-reading):
1. `syms = if args.symbols.is_empty() { held symbols from super::positions(cfg)? } else { args.symbols.clone() }`.
   (Extract the `symbol` field from each positions row.)
2. `let client = super::connect(cfg)?;`
3. for each sym: `let contract = Contract::stock(sym).build();`
   `let data = client.historical_data(&contract, BarSize::Day).what_to_show(WhatToShow::Trades)
   .duration("2 Y".to_duration()).fetch().map_err(|e| AppError::data(.., "sma-signal"))?;`
   `let bars: Vec<Bar> = data.bars.iter().map(|b| Bar { ym: ym_of(&b.date), close: b.close }).collect();`
   `let sig = sma_signal(&bars, args.sma);`
4. Return `json!({ "signals": [ per-symbol {symbol, state:as_str, as_of:"YYYY-MM", month_end_close, sma,
   distance_pct, latest_close, latest_sma, latest_distance_pct, bars_used} ] })`.

`ym_of(&BarTimestamp) -> (i32,u32)` (module-private helper): match `BarTimestamp::Date(d) => (d.year(),
u8::from(d.month()) as u32)` and `BarTimestamp::DateTime(dt) => (dt.year(), u8::from(dt.month()) as u32)`.
`use ibapi::market_data::historical::{BarSize, BarTimestamp, Duration, ToDuration, WhatToShow};` (mirror
`history.rs`), `use ibapi::prelude::Contract;`.

## Review rejection 01 -- 2026-07-07 -- codex

historical bar set, the pure function's normal guards do not fire for `n == 0`, and `sma_at` slices
`bars[start..=i]` with `start == i + 1`, causing a panic instead of the repo's structured JSON error
envelope. Fix by rejecting zero at the public command boundary (config/usage error before gateway work)
and making the pure seam total for `n == 0` as a defensive guard. Keep `tests/sma_signal.rs` untouched;
freeze gate already passed for PR #31.
