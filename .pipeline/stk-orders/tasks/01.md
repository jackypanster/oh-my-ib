## Scope (impl-paths only — do NOT touch `tests/`)

- `src/cli.rs`: `Buy(OrderArgs)`, `Sell(OrderArgs)`, `Cancel(CancelArgs)`;
  `OrderArgs { symbol: String, quantity: f64, #[arg(long)] limit: Option<f64> }`;
  `CancelArgs { order_id: i32 }`. Docs: buy "Place a BUY order (paper by default; live needs
  --live + OMI_ALLOW_LIVE=1)", sell/cancel analogous.
- `src/ib/trade.rs` (NEW — the ONLY file with write calls):
  - `pub fn build_stk_order(symbol: &str, side: Action, quantity: f64, limit: Option<f64>)
    -> (Contract, Order)` — exactly per arch.md (LMT/MKT, TIF Day, Contract::stock).
  - `pub fn shape_order_ack(order_id: i32, status: &str, symbol: &str, action: &str,
    quantity: f64, limit_price: Option<f64>) -> Value` — exact 6-key object.
  - `pub fn require_live_write_gate(cfg: &Config) -> Result<(), AppError>` — effective-port
    rule (`cfg.port == LIVE_PORT` && env `OMI_ALLOW_LIVE` != "1" ⇒ `AppError::config`), runs
    BEFORE `super::connect`.
  - `pub fn buy(cfg, args)` / `pub fn sell(cfg, args)`: validation (quantity > 0, limit > 0
    when present ⇒ else `AppError::usage`) → gate → connect → `next_valid_order_id()` →
    `build_stk_order` → `place_order(id, &contract, &order)` → bounded first-ack loop
    (`timeout_iter_data(TAKE_FIRST_TIMEOUT)` + Instant-classified None, ADR 0016 pattern):
    first `PlaceOrder::OrderStatus` (Debug-render `.status`) or `PlaceOrder::OpenOrder`
    (Debug-render `.order_state.status`) is the ack; skip ExecutionData/CommissionReport;
    timeout/instant-end ⇒ `AppError::timeout` naming the ALLOCATED order id + "may have been
    SUBMITTED — verify with \`omi orders\`; do NOT retry blindly" (context "buy"/"sell").
    Success ⇒ `shape_order_ack`.
  - `pub fn cancel(cfg, args)`: gate → connect → `cancel_order(order_id, "")` → bounded first
    `CancelOrder::OrderStatus` ⇒ `json!({"order_id", "status"})`; timeout ⇒ same UNKNOWN
    posture (context "cancel").
  - NO retry logic anywhere. NO write calls outside this file.
- `src/ib/mod.rs`: `mod trade;` +
