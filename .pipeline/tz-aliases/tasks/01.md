# Card 01 — auto-register timezone aliases

Make `tests/tz_aliases.rs` green and let `omi` connect to the Tiger/HK gateway without
`IBAPI_TIMEZONE_ALIASES`.

## Scope
- `src/tz.rs`:
  - `pub fn builtin_aliases() -> &'static [(&'static str, &'static str)]` — curated unambiguous set:
    `("HKT","Asia/Hong_Kong"), ("JST","Asia/Tokyo"), ("KST","Asia/Seoul"), ("SGT","Asia/Singapore")`.
  - `pub fn register_builtin_aliases()` — `std::sync::Once`-guarded; for each entry call
    `ibapi::register_timezone_alias(abbr, iana)`.
- `src/lib.rs`: add `pub mod tz;`.
- `src/ib/mod.rs`: call `crate::tz::register_builtin_aliases()` as the first line of `connect`.

## Freeze coverage
Frozen (`tests/tz_aliases.rs`): `builtin_aliases()` contains `HKT→Asia/Hong_Kong`, table non-empty,
no empty entries, unique keys. NOT frozen — **live acceptance**: `unset IBAPI_TIMEZONE_ALIASES;
omi --live health` connects to the live Tiger gateway (needs the gateway up; gateway-dependent).
