# Card 02 — six read subcommands wired to ibapi

Make `tests/data_commands.rs` green. Builds on card 01's crate (do card 01 first).

## Scope (each = connect → request → model struct → output::emit)
- `account`   → `AccountSummary` (net_liquidation, total_cash, buying_power, available_funds, currency).
- `positions` → `[Position]` (symbol, conid, qty, avg_cost, market_value, unrealized_pnl).
- `orders`    → `[OpenOrder]` (open/working orders only — READ, never place).
- `quote <SYMBOL>` → `Quote` with a `delayed: bool` field reflecting `--md-type`; set market-data type
  (delayed=3, live=1, frozen=4) before the snapshot; take first snapshot then drop the subscription.
- `contract <SYMBOL>` → `ContractInfo` (conid, exchange, currency, sec_type) via `contract_details`.
- `history <SYMBOL> --bar <1d|1h|...> --duration <1M|...>` → `[Bar]` via `historical_data`.
- `quote --help` MUST mention `--md-type`; `history --help` MUST mention `--bar` and `--duration`.
- Map every `ibapi::Error` → `AppError` (connection vs not_found vs data) — never bubble a raw error.
- Add `#[cfg(test)] mod tests` (impl-paths) for the model→JSON shaping (white-box, e.g. quote carries
  the `delayed` flag).

## Review rejection 01 — 2026-06-28
- `src/ib/account.rs`: output is raw `account_summary` keyed by IB tag names, but PRD/arch require stable agent-facing fields: `account`, `net_liquidation`, `total_cash`, `buying_power`, `available_funds`, `currency`.
- `src/ib/account.rs` / `src/ib/positions.rs` / `src/ib/orders.rs`: `--account`/`cfg.account` is ignored; multi-account runs can return all accessible accounts despite a requested account filter.
- `src/ib/positions.rs`: output lacks `qty`, `market_value`, and `unrealized_pnl`, and uses `position`/`average_cost` instead of the card's `qty`/`avg_cost` names.
- `src/ib/orders.rs` / `src/ib/history.rs`: successful `orders`/`history` serialize Rust `Debug` strings, not stable JSON objects.
- Required retry: shape account/positions/orders/history JSON to the documented contract and honor account scoping. If ibapi `positions()` cannot provide market value / unrealized PnL, route back to `pipeline-task` to re-spec honestly instead of merging a shape that paper acceptance cannot satisfy. See `.pipeline/phase1-readonly/reviews/review-01.md` and `review-02.md`.
