# Card 01 — retry transient connect errors

Make `tests/connect_retry.rs` green and make `ib::connect` resilient to transient EAGAIN.

## Scope (all in `src/ib/mod.rs`)
- `pub fn is_transient_io(kind: std::io::ErrorKind) -> bool` → `matches!(kind, WouldBlock | Interrupted | TimedOut)`.
- `connect`: bounded retry loop. `const MAX_CONNECT_RETRIES: u32 = 3; const BACKOFF_MS: u64 = 250;`
  On `Err(err)` where `matches!(&err, ibapi::Error::Io(e) if is_transient_io(e.kind()))` and
  `attempt < MAX_CONNECT_RETRIES`: `attempt += 1; std::thread::sleep(Duration::from_millis(BACKOFF_MS * attempt as u64)); continue`.
  Otherwise map to `AppError::connection(...)`. Keep `crate::tz::register_builtin_aliases()` first.

## Freeze coverage
Frozen (`tests/connect_retry.rs`, std-only): `is_transient_io` true for WouldBlock/Interrupted/TimedOut,
false for ConnectionRefused/NotFound/PermissionDenied. NOT frozen — reviewed + live acceptance: that the
retry smooths back-to-back `account_updates` (gateway-dependent; EAGAIN is intermittent, so acceptance =
`omi --live account` then immediately `omi --live positions` succeeding across several runs). Dead-port
(refused) must still fail fast — keeps tests/cli_contract.rs::connection_error_is_json_envelope fast.
