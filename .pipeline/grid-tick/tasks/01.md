- `pub fn plan_grid_tick(cfg: &GridConfig, acct: &AccountSnap, positions: &std::collections::HashMap<String, PositionLite>, open: &[OpenOrderLite]) -> Vec<Action>` — the PURE reconcile (see algorithm below).

**Planner algorithm** (ADR 0033 D-PLANNER; `round2(x) = (x*100.0).round()/100.0`):

```
let mut cancels = vec![]; let mut places = vec![];
for s in &cfg.symbols {
    let pos_qty = positions.get(&s.name).map(|p| p.qty).unwrap_or(0.0);
    let existing: Vec<&OpenOrderLite> = open.iter().filter(|o| o.symbol == s.name).collect();
    // desired: 0..2 (side, qty, limit)
    let mut desired: Vec<(Side, f64, f64)> = vec![];
    if pos_qty > 0.0 {
        let avg = positions[&s.name].avg_cost;
        desired.push((Side::Sell, pos_qty.min(cfg.lot as f64), round2(avg * (1.0 + s.rise_pct/100.0))));
        let floor = (cfg.cash_floor_pct/100.0) * acct.net_liquidation;
        let buy_ok = acct.total_cash >= floor && pos_qty + (cfg.lot as f64) <= (s.max_shares as f64);
        if buy_ok {
            desired.push((Side::Buy, cfg.lot as f64, round2(avg * (1.0 - s.drop_pct/100.0))));
        }
    }
    // reconcile: keep an existing order iff same side & qty (±1e-6) & |Δlimit|<=0.005; else place the desired.
    let mut kept: std::collections::HashSet<i32> = HashSet::new();
    for (side, qty, limit) in &desired {
        match existing.iter().find(|o| o.side == *side && (o.qty - qty).abs() < 1e-6
                                      && (o.limit - limit).abs() <= 0.005 && !kept.contains(&o.order_id)) {
            Some(o) => { kept.insert(o.order_id); }               // keep — no action
            None => places.push(Action::Place { symbol: s.name.clone(), side: *side, qty: *qty, limit: *limit }),
        }
    }
    for o in &existing { if !kept.contains(&o.order_id) { cancels.push(Action::Cancel { order_id: o.order_id }); } }
}
cancels.into_iter().chain(places).collect()   // ALL Cancels first, then ALL Places (global)
```

Symbols NOT in `cfg.symbols` are never iterated ⇒ their orders/positions are invisible (blast-radius guard).

### 4. `src/ib/grid.rs` (NEW) + `mod grid; pub use grid::grid_tick;` in `src/ib/mod.rs` — the driver

`pub fn grid_tick(cfg: &Config, args: &GridTickArgs) -> Result<Value, AppError>`:
1. `let gcfg = crate::grid::GridConfig::load(&args.config)?;`
2. `if cfg.port == LIVE_PORT { return Err(AppError::config("grid-tick is paper-only in v1 — use paper :4002", "grid-tick")); }` (offline, before connect).
3. `let client = super::connect(cfg)?; let account = super::resolve_account(&client, cfg)?;`

   → map its JSON rows into `Vec<OpenOrderLite>` (parse `action` "Buy"/"Sell"→Side, skip/keep per symbol;
   `order_id`, `symbol`, `limit_price`→limit, `quantity`→qty). Orders whose action isn't Buy/Sell ⇒ skip.
6. `let actions = crate::grid::plan_grid_tick(&gcfg, &snap, &positions, &open);`
7. If `args.dry_run`: return the envelope with `dry_run:true` and the planned actions (no execution).
8. Else execute in order (Cancels precede Places already): `Cancel` → `trade::cancel_with_client(&client, id)`;
   `Place` → `let (contract, order) = trade::build_stk_order(&symbol, ib_action, qty, Some(limit));`
   then `trade::place_with_client(&client, "grid-tick", &contract, &order, &account, ack)`. **Stop on the
   first error** (record it, return partial — ADR 0017 no blind retry; the next tick re-reconciles).
9. Return `{ "account": account.0, "dry_run": <bool>, "planned": actions.len(), "actions": [<results>] }`.

## Review rejection 01 -- 2026-07-07 -- codex

REJECT: `src/ib/grid.rs` resolves one account for positions and placements but reads open orders with
`cfg.account.as_deref()`. Because `open_orders_with_client` filters only when this value is `Some`, a
no-`--account` grid tick on a multi-account gateway can ingest and cancel another account's open order for
a configured symbol. Fix the driver to pass the resolved account to `open_orders_with_client` (for example
`Some(account.0.as_str())`) so read-side reconciliation, positions, and placements all scope to the same
single resolved account. Keep `tests/grid_tick.rs` untouched; freeze gate already passed for PR #30.
